Producing .EXE Files:

In addition to the macro example given in the source tree for NASM, I give a -fbin form here. The EXE Header is crafted as the example code shows. This fits the method used, as the code manages its own segments. The EXE Header is made to be 100h bytes in size so that when the loader strips it off and prepends a Program Segment Prefix, the code and data alignments following it are unaffected. The end of the Header is the initial temporary stack, defined by entries in the Header Record itself. Winxx, XP's CMD.EXE won't load an -fbin .COM if > 64k, yet the same code with this .EXE Header, will load and can run.
